# NeuraLife — E-Library

*The complete design specification for the E-Library — the content reading, interaction, and learning experience layer delivered on SmartPad, Mobile, and Tablet. This document is the engineering and design reference for building the E-Library on all three surfaces.*

---

## 1. What the E-Library Is

The E-Library is the student-facing layer of the Content Layer. It is where PRAJNA Framework content is experienced. It is not a PDF viewer or a video player — it is a structured, interactive, AI-connected learning environment.

Every topic in the E-Library is experienced as a **segmented journey**: 12 screens, one per segment, each building on the last. The student moves through the topic at their own pace, with an AI chat assistant available on every screen, audio on every screen, notes on every screen, and a quiz gate at the end that earns NeuraCoins.

The E-Library runs fully offline. All content for the student's current grade is downloaded on first login. Interaction, quiz, and progress data is queued and synced when online.

## 2. Chapter Entry — Four Content Modes

When a student opens a chapter, they see four tabs:

```
┌──────────────────────────────────────────────────────┐
│  Chapter 3 — Quadratic Equations                     │
├────────────┬───────────┬─────────────┬───────────────┤
│ 📖 Textbook │ 📝 Workbook │ 🗒 Classwork │ 📋 Homework  │
└────────────┴───────────┴─────────────┴───────────────┘
```

| Tab | Content | Who manages it |
|-----|---------|----------------|
| **Textbook** | 12 PRAJNA segments for each topic — generated by Content Agent, audited and approved | NeuraLife Content Team |
| **Workbook** | Practice problem sets + additional exercises beyond the 10 in the Textbook tab | NeuraLife Content Team |
| **Classwork** | Notes taken by the student while reading (rough notes, saved per topic) | Student |
| **Homework** | Problems or notes assigned by the teacher to this class for this chapter | Teacher via Teacher App |

Default tab on open: **Textbook**.

The Workbook and Homework tabs show a list of topics with assigned content. Tapping a topic opens the relevant content or problem set.

The Classwork tab shows a list of topics for which the student has saved notes. Tapping opens the notes viewer for that topic.

## 3. Topic Navigation Model

### Entry

Student taps a topic from the chapter topic list. They land directly on **Segment 1 — Concept Summary**, in their chosen language.

### Segment-by-Segment Flow

Each topic is 12 segments. Each segment occupies its own **full screen**. There is no scrolling within a segment — the content fits the screen. If a segment has more content than fits (e.g., a long concept explanation), it is paginated internally with a subtle down-arrow indicator.

**Progress bar:**
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 Segment 3 of 12  ■■■□□□□□□□□□
                  Concept Explanation
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

- Shown at top of every segment screen
- Each completed segment shown as filled block, current as active, pending as empty
- Tapping a completed block navigates back to that segment
- Tapping a pending block is disabled (must complete in order on first pass; free navigation after topic completion)

### Segment Order

```
1.  Concept Summary
2.  Concept Explanation
3.  Key Terms
4.  Did You Know
5.  Interaction (Play-to-Learn)
6.  Prerequisites
7.  Audio Text (TTS listen-along)
8.  SVG Diagram
9.  CSS Diagram (LCD only; replaced by enhanced SVG on E-Ink)
10. Free Style (infographic / formula card / timeline)
11. YouTube Recommendations
12. Problems
```

*Note: Prerequisites segment is hidden if no prerequisites exist for this topic (DB returned empty list).*
*Note: CSS Diagram segment is hidden on E-Ink SmartPad.*

## 4. Segment Screen Anatomy

Every segment screen follows the same shell:

```
┌─────────────────────────────────────────────────────────┐
│ ← Back    Quadratic Equations    ■■■□□□□□□□□□   🎵 🗒   │  ← Header bar
├─────────────────────────────────────────────────────────┤
│                                                         │
│                  SEGMENT CONTENT                        │  ← Full-screen content area
│                                                         │
│                                                         │
│                                                         │
├─────────────────────────────────────────────────────────┤
│  [💬 Ask AI]          [✓ Understood]         [Next →]   │  ← Action bar
└─────────────────────────────────────────────────────────┘
```

**Header bar elements:**
- Back arrow (← navigates to previous segment or chapter list)
- Topic title (truncated with ellipsis if long)
- Progress bar (12 segment blocks)
- 🎵 Audio button (opens TTS for this segment)
- 🗒 Notes button (opens notes overlay for this segment)

**Action bar — three buttons:**

| Button | Behaviour |
|--------|-----------|
| 💬 **Ask AI** | Opens AI Chat overlay (see Section 6). Does not mark segment as complete. |
| ✓ **Understood** | Marks this segment as completed (green checkmark on progress bar). Moves to next segment. |
| **Next →** | Moves to next segment WITHOUT marking as completed. Progress bar block remains unfilled. |

Design rule: **Understood** button is primary (prominent). **Next** is secondary (lighter weight). The student must consciously choose to skip marking a segment understood — the default is to mark it.

If the student completes all 12 segments, the last screen transitions to the **Topic Quiz** (see Section 12).

## 5. Segment-Specific Rendering Rules

### Segment 1 — Concept Summary
Full-screen, centred layout. Large font (24px+). The one-sentence summary displayed prominently. Below it: topic name + subject + grade in muted text. A subtle background illustration (subject-specific watermark, very low opacity) fills the screen.

### Segment 2 — Concept Explanation
Structured reading experience. Rendered as markdown with:
- Headers for each structural section (Hook / Intuition / Definition / Example / Diagram connection / Exam tip)
- Key terms bolded and **tappable** — tap opens inline definition card from key_terms segment (see Section 9.1)
- Smooth internal pagination if content overflows screen height

### Segment 3 — Key Terms
Card-based layout. Each term is a card:
```
┌──────────────────────────────────┐
│  వర్గ సమీకరణం                   │
│  Quadratic Equation              │
│  ─────────────────────────────── │
│  An equation where the highest   │
│  power of x is 2.                │
│                                  │
│  Example: In a 65×30 metre       │
│  paddy field, if x² + 5x = 150  │
│  describes the width — that's a  │
│  quadratic equation.             │
└──────────────────────────────────┘
```
Scrollable list of 6–10 cards. Swipe-to-advance through cards as an option.

### Segment 4 — Did You Know
Full-screen card. Accent colour background (#f59e0b tinted). Large quote-style typography. Source cited at bottom if available. Single focused screen — nothing else.

### Segment 5 — Interaction
See Section 10 (full interaction design spec).

### Segment 6 — Prerequisites
Shown only if prerequisites exist. List of prerequisite topics with:
- Grade badge (e.g., "Class 8")
- Topic name
- Subject badge
- **"Review Now"** button → opens that topic in E-Library (same navigation stack, Back returns here)
- **"Add to Queue"** button → adds to Prerequisite Task Queue (see Section 15)

If prerequisites list is empty: segment is skipped silently (counted as auto-completed).

### Segment 7 — Audio TTS
See Section 8 (full audio design spec).

### Segment 8 — SVG Diagram
See Section 9 (full diagram design spec).

### Segment 9 — CSS Diagram (LCD only)
Same shell as Segment 8. Only shown on Mobile and Tablet (LCD surfaces). On SmartPad E-Ink, this segment is auto-skipped (student never sees a blank segment — it simply does not appear in the progress bar on SmartPad).

### Segment 10 — Free Style
Full-screen rendered HTML. Responsive. No outer chrome — the HTML fills the content area edge-to-edge. Zoom-in supported via pinch. Full-screen toggle button (⛶) in top right.

### Segment 11 — YouTube Recommendations
List of 3–4 video cards. Each card:
```
┌────────────────────────────────────────────┐
│ ▶  Quadratic Equations (Telugu)            │
│    Telugu Maths — 9 min                    │
│    [☐ Watched]  [Open in YouTube ↗]       │
└────────────────────────────────────────────┘
```
"Watched" checkbox is student-side only (not tracked as completion). Opening YouTube launches the YouTube app or browser (requires network).

### Segment 12 — Problems
See Section 11 (full problem segment spec).

**Design rules:**
- Slides up as a bottom sheet (70% screen height). Background content visible but dimmed.
- X button closes the overlay. Student returns exactly where they were on the segment.
- Prebuilt questions are generated per topic (5–6 questions tailored to the topic, not generic).
- Free-text input available after selecting a prebuilt question or directly.
- Chat history persists within the session. Cleared when student exits the topic.

**Backend context sent to agent:**
```json
{
  "board": "SCERT_AP",
  "grade": 10,
  "subject": "MATHEMATICS",

  "segment_content": "...current segment text...",

The AI responds in the student's chosen medium (Telugu or English). Responses are grade-appropriate — the agent is briefed on the age band.

**Quick Recap:** If the student asks "explain this again" or "give me a summary", the AI produces a 3-bullet point recap of the topic — this is the Quick Recap feature available via chat.

## 7. Notes System

### Rough Notes Overlay

Triggered by the 🗒 button in the header bar. Opens a full-screen overlay on top of the segment.

On **SmartPad**: the overlay is a blank page. The stylus is active. The notification bar shows: `[Topic Name] — Notes | [✓ Save] [✗ Discard]`. The student writes freely with the stylus. No bounding box — the entire screen below the notification bar is the writing surface. Handwriting is saved as an image (HWR-1 does NOT process these notes — they are freeform rough notes, not assessed writing).

On **Mobile/Tablet**: the overlay shows a lined text area with optional stylus input.

### Notes Storage

Notes are saved per segment (e.g., "Concept Explanation notes for Quadratic Equations Introduction"). They are accessible from:
- The **Classwork** tab of the chapter
- A dedicated **My Notes** section in the app
- The student profile page

Notes are local-first (stored on device) and synced to cloud when online.

### Relationship to Classwork Tab

Every rough note saved from a segment appears in the Classwork tab → that topic → that segment. This is the student's personal digital exercise book. The teacher cannot read these notes (they are private to the student).

## 8. Audio TTS + Word Highlighting

### Trigger

🎵 icon in the header bar. Always visible on every segment.

### Behaviour by Band

| Band | Default | Highlighting |
|------|---------|-------------|
| Foundation (1–3) | Auto-plays on topic open | Mandatory word-by-word highlighting |
| Elementary (4–6) | Auto-plays on topic open | Mandatory word-by-word highlighting |
| Middle (7–8) | Manual tap to play | Optional — student can toggle |
| Secondary (9–10) | Manual tap to play | Optional — student can toggle |

### Controls

When audio is playing:
```
┌────────────────────────────────────────┐
│  🎵  ██████████████░░░░░  2:14 / 3:45  │
│       ⏮ 10s  ⏸ Pause  ⏭ 10s  1×▾     │
└────────────────────────────────────────┘
```
- **⏮ 10s / ⏭ 10s**: Rewind/forward 10 seconds
- **Pause/Resume**: State icon changes
- **1×▾**: Playback speed selector (0.75×, 1×, 1.25×, 1.5×)

### Tap-to-Start-From

When audio is paused or stopped: the student can **tap any word or sentence** in the segment text. The audio starts reading from the beginning of that sentence. A blue underline appears under the tapped sentence to confirm.

### Word-by-Word Highlighting

Each word is wrapped in a span. As TTS progresses, the current word gets a yellow highlight background (like a physical highlighter). Previous words fade to muted colour. This is especially critical for Foundation/Elementary bands where students are building reading fluency.

### Language

Audio plays in the student's chosen medium. Telugu TTS uses the configured voice (to be decided — see Open Questions). The audio_text field generated by Claude is the exact script — no markdown, equations spelled out in words.

## 9. Diagram Experience

### SVG Diagram (Segments 8)

**Default view:** Full-width diagram centred in the content area, below a caption.

**Full-screen mode:** Student taps the diagram or a ⛶ expand icon. Diagram fills the screen. Background is white (E-Ink) or white (LCD). Header bar collapses to show only ← Back.

**Zoom:** Pinch-to-zoom available in both normal and full-screen mode. Double-tap to fit.

**Tap Targets:**

When the student taps a tap target (`<g id="tap_N">` element in the SVG):
- A **bottom sheet panel** slides up from the bottom (40% of screen height)
- The diagram behind it dims to 60% opacity
- Panel shows:
  ```
  ┌──────────────────────────────────────────┐
  │  Root of the equation           [  X  ]  │
  │  ─────────────────────────────────────── │
  │  The point where the parabola crosses    │
  │  the x-axis. At this point, y = 0.       │
  │                                          │
  │  Formula: x = −b ± √(b²−4ac) / 2a       │
  │                                          │
  │          [See in Key Terms →]            │
  └──────────────────────────────────────────┘
  ```
- Student can tap a **different tap target** while panel is open — panel updates to new target without closing
- Tap outside panel or X to close
- "See in Key Terms →" navigates to Segment 3, scrolled to that term

### CSS Diagram (Segment 9, LCD only)

Same shell as SVG. Animation state:
- **Auto-plays once** when the student first arrives on the segment
- A **⏸/▶ toggle icon** (bottom-right corner) lets student pause/resume the animation
- After one full play, animation pauses at the final state

### E-Ink SmartPad: SVG as Textbook Diagram Replacement

On E-Ink, the SVG diagram must be the quality equivalent of a printed textbook diagram — not an illustration, a precise educational visual. Requirements:
- All relevant labels visible at default size
- Structural clarity: no ambiguous lines
- Tap interactions work via stylus tap
- Caption always visible below diagram

## 10. Play-to-Learn Interactions (Stylus-First Design)

**Design principle:** The SmartPad exists because stylus interaction develops cognitive muscle memory for writing. Every interaction type should maximise stylus use where it genuinely aids learning. Stylus must never feel forced — it must feel natural and superior to touch for the task.

### 10.1 Tap-to-Sequence

**Engagement design:**
- Cards displayed in shuffled order in a 2-column grid (or single column if many cards)
- Student taps cards in the correct order with stylus
- Tapped card: scale 1.05 → lifts with subtle shadow → moves to a "sequence zone" at the top of the screen with its sequence number
- Wrong order: that card shakes horizontally (300ms) + brief red tint → returns to pool
- After 2 consecutive wrong taps: a ghost outline appears in the sequence zone showing where the next correct card belongs (size hint, not identity hint)
- After 3 wrong taps: hint text appears: "Think about what happens first in the process..."
- Completion: all cards snap into final sequence with a green left-to-right wave animation, confetti micro-animation, and a brief satisfying sound cue

**Stylus use:** Stylus tap (not touch) preferred. Hover detection (if device supports) shows a faint lift shadow before the student taps — previewing the action.

### 10.2 Label-the-Diagram

**Engagement design:**
- Diagram shown at 80% screen width. Label chips shown at bottom in a scrollable horizontal list.
- **Stylus tap label** → label selected (blue pulsing border)
- **Stylus tap position on diagram** → label placed. A dotted leader line from label chip to position.
- Correct placement: label snaps to correct anchor, green flash + expand animation. Leader line turns solid.
- Wrong placement: red flash + label chip bounces back to the list
- After 2 wrong attempts on the same label: dotted circle appears on diagram showing the correct zone (zone hint, not exact point)
- Completion: all labels placed → diagram "comes alive" with a subtle ripple from each label

**Stylus use:** Both steps (select + place) use stylus tap. Touch is a fallback only.

### 10.3 Slider-Parameter

**Engagement design:**
- Slider at bottom. Variable name + current value displayed prominently beside the slider.
- Graph/diagram updates **in real time** as student drags the slider.
- At key threshold values (e.g., resistance doubles, velocity equals escape velocity): a **pulse flash** on the relevant part of the graph + a brief label that appears for 2 seconds: "Resistance doubled — current halved"
- Student can tap on the graph to set the slider to the value at that graph position (bi-directional)
- After student interacts for 30 seconds: prompt appears: "What happens when you move the slider to its maximum?"

**Stylus use:** Slider drag with stylus. Value tap on graph with stylus tap.

### 10.4 Stylus-Fill-Equation

**Engagement design:**
- Problem statement shown at top (full width, clear typography).
- Below: a set of **equation steps**, each with a blank bounding box for stylus writing.
- The student writes the answer/step in the bounding box with the stylus.
- Notification bar shows: `Step 2 of 4  |  ✓ Check  |  💡 Hint`
- HWR-1 reads the stylus input. When student taps ✓ Check:
  - If correct: bounding box flashes green, next step unlocks
  - If incorrect: bounding box flashes red. The **first incorrect character or term** is highlighted in red, others dimmed. Student can erase and retry.
- After 2 failed attempts on a step: Hint button pulses. Tapping hint reveals the method (not the answer) for that step.
- HWR-1 confidence popup: if HWR-1 confidence < 0.75, a popup shows: "Did you write: [HWR-1 result]?" with [Yes / No, let me rewrite] options. This data is used for HWR-1 model training.

**Bounding box design:**
- Fixed-height box (approximately 2–3 lines of writing space)
- Light blue border (active), green (correct), red (wrong)
- Light lined background (like graph paper) for writing guidance
- Stylus pressure shows ink weight — makes writing feel natural

**Time tracking:** SHE-1 records time per step silently. No visible countdown. No pressure.

### 10.5 Time Limits

**Learning mode (default):** No time limit. SHE-1 records time-on-task silently. The student learns at their own pace.

**Challenge mode (optional, future):** A subtle timer bar (not a countdown clock) depletes slowly at the top of the screen. No alarm — when time ends, interaction submits with current progress. Used only for competitive challenges, not the standard problems segment.

When student taps **[Solve]**:

├─────────────────────────────────────────────────────────┤

│                                                         │
├─────────────────────────────────────────────────────────┤
│  ┌───────────────────────────────────────────────────┐  │
│  │                                                   │  │
│  │     [Student writes here with stylus]             │  │  ← Bounding box
│  │                                                   │  │  (fills remaining screen)
│  │                                                   │  │
│  └───────────────────────────────────────────────────┘  │
│                                                         │
│  [✗ Clear]                              [✓ Check]       │
└─────────────────────────────────────────────────────────┘
```

**On Mobile/Tablet (non-stylus solve):** MCQ or typed input instead of stylus, with a "Send to SmartPad" button that creates a due task on the student's linked SmartPad.

**Send to SmartPad flow:** If student is accessing E-Library on Mobile/Tablet but wants to solve with stylus:
- Tap "Solve on SmartPad" → problem is added to a **SmartPad Due Tasks** queue
- On SmartPad, the student sees a notification: "3 problems waiting from Mobile"
- Solves on SmartPad, scores are synced back

### Hint System

- **Notification bar**: "💡 Hint (−2 coins)" — student sees the cost before tapping
- Hint 1 (−2 coins): Conceptual nudge
- Hint 2 (−3 coins): Method pointer
- Hint 3 (−5 coins): Near-solution (no final answer)
- Total hint cost: max −10 coins per problem
- Alert shown before first hint: "Using hints reduces your score by a small amount. Your learning matters more than your score — use them freely."

### Wrong Step Detection

When HWR-1 + GAP-1 processes the student's work:
- The **first incorrect step** in the written solution is highlighted with a red underline
- Correct steps remain visible, dimmed
- Prompt: "Your first step looks right. Check Step 2."
- Student can erase from that point and continue
- On 3rd wrong attempt: solution reveals step-by-step, one step at a time, with the student tapping "Show next step" to reveal each

### Submitting

Student taps **Submit for Assessment** from the problem list.
- If any problems are unattempted: prompt "You've skipped 3 problems. Submit anyway or solve them first?" [Submit] [Go back]
- Score calculated: each problem scored 0–10 based on correctness + hints used
- Final score shown as a percentage: Foundation 30% + Standard 50% + Advanced 20% weighted

## 12. Topic Completion, Quiz, and NeuraCoins

### Completion Flow

After the student marks all 12 segments as **Understood** (or skips some with Next), the final segment's action bar shows:

```
[💬 Ask AI]          [✓ I'm Ready — Take Quiz]
```

The "Next" button is replaced by the quiz call-to-action.

### Topic Quiz

The quiz is drawn from the **Standard** tier of the Problems segment (5 questions). It is presented as 5 questions, each on its own screen:

- No hints during the quiz
- No coin deduction tracking (quiz is pass/fail only)
- Student submits each answer before seeing the next
- Time is not shown — SHE-1 records it

**Pass condition: 70%+ (4 out of 5 correct)**

### On Pass (≥70%)

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│           🎉  Topic Complete!  🎉                       │
│                                                         │
│    Quadratic Equations — Introduction                   │
│                                                         │
│         ⭐⭐⭐⭐⭐   Score: 88%                          │
│                                                         │
│         💰  +20 NeuraCoins earned!                     │
│              (Total: 340 coins this term)               │
│                                                         │
│  [Share to NeuraSphere ↗]                              │
│                                                         │
│  ─────────────────────────────────────────────         │
│  [🚀 Next Topic: Standard Form →]   [🏠 Home]          │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

Celebration animation: confetti drops, coins animate from bottom to the coin balance indicator. The score number counts up.

### On Fail (<70%)

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│         📖  Let's Review Again                         │
│                                                         │
│    Score: 54%  (Need 70% to pass)                      │
│                                                         │
│    The segments have been reset.                        │
│    Complete all segments again to unlock the quiz.     │
│                                                         │
│    Where students usually struggle:                    │
│    • Factoring with negative signs                     │
│    • Identifying which method to use                   │
│                                                         │
│  [💬 Ask AI about what went wrong]  [Start Again →]   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

On fail: **all 12 segments are unmarked** in the progress bar. The student must complete all segments again. This is intentional — a second pass after seeing where they went wrong typically yields much better retention. GAP-1 signals from the failed quiz are used to personalise the "Where students usually struggle" hint.

### NeuraCoins Earning Schedule

| Achievement | Coins |
|-------------|-------|
| Topic quiz pass (70–84%) | 10 |
| Topic quiz high pass (85–99%) | 20 |
| Topic quiz perfect (100%) | 30 |
| All problems solved in problem segment (before quiz) | 5 bonus |
| Chapter completion (all topics passed) | 30 bonus |
| 7-day consecutive study streak | 25 bonus |
| 30-day consecutive study streak | 75 bonus |
| Unit Test / FA Test — top quartile (entered by teacher) | 50 / 75 |

NeuraCoins balance resets at academic year end. Students must redeem before year close.

## 13. NeuraSphere Achievement Sharing

## 15. Prerequisite Surface Flow

When GAP-1 detects a PREREQUISITE_GAP error during problem solving:

```
┌────────────────────────────────────────────────────────┐
│  🔍  Gap Detected                                      │
│  ─────────────────────────────────────────────         │
│  Your answer shows a gap in:                           │
│  "Factoring — Class 8, Chapter 4"                      │
│                                                        │
│  Mastering this will help you solve this problem.      │
│                                                        │
│  [📖 Study Now]     [+ Add to My Queue]   [Skip]       │
└────────────────────────────────────────────────────────┘
```

**Study Now:** Opens the prerequisite topic in E-Library (same navigation stack). When student completes that topic (or taps Back), they return to the original problem.

**Add to My Queue:** Adds to the Prerequisite Task Queue — a dedicated section in the student's home screen showing "Prerequisite gaps to address."

**Skip:** Dismisses for this session. GAP-1 still records the pattern. Teacher sees the gap flag.

The prerequisite topic for a student becomes a "recommended" item that appears in the student's personalised feed, surfaced again before the next session on this subject.

## 16. Offline Architecture

### First Login

When the student logs in for the first time on any device:
1. System detects the student's current enrolled grade and board
2. A progress screen shows: "Downloading your textbooks… (1.2 GB)"
3. All content bundles (`.nlc` files) for the current grade, all subjects, are downloaded
4. Download happens in the background after the initial progress screen clears — student can use the app while it continues

### Session Caching for Other Grades

If the student navigates to a topic from a different grade:
1. A download prompt: "This content (45 MB) will be downloaded for this session. Download now?"
2. On confirm: downloads and caches for the session duration
3. On app close: cached content from other grades is cleared (storage management)
4. On next open: prompts again if student revisits

### Offline Sync Queue

When offline, all student actions are queued locally:
- Segment completion events
- Problem attempt data
- Quiz scores
- Notes saved
- NeuraCoins earned

On network restore: queue syncs in the background. Data is never lost.

### Storage Budget (Student Device)

| Content | Size |
|---------|------|
| Current grade, all subjects (`.nlc` bundles) | ~60–150 MB |
| Student progress database | ~5 MB |
| Notes (handwriting images) | Variable (~1 MB per chapter of notes) |
| **Total reserved** | **~300 MB** |

## 19. Confirmed Decisions

| Decision | Rationale |
|----------|-----------|
| One segment per full screen (no scrolling across segments) | Reduces cognitive overload. Each screen has one purpose. Student knows exactly where they are. |
| Three actions per segment: Ask AI / Understood / Next | Gives student agency. "Understood" is primary — celebrates confident completion. "Next" allows progression without forcing false confidence. |
| Progress bar shows all 12 segments; tapping back to completed segments allowed | Reinforces the journey. Students can review any completed segment freely. |
| Fail = all segments unmarked, redo from beginning | Spaced repetition effect. Second pass after seeing quiz gaps yields significantly better retention. |
| Stylus-first interaction design on SmartPad | The SmartPad's value proposition is handwriting. Every interaction that can use stylus, should. |
| Problem statement fixed at top; bounding box below | Student never loses sight of what they're solving. Mirrors physical exam paper behaviour. |
| No time limit in learning mode | Pressure is antithetical to deep learning. SHE-1 tracks time silently for analytics only. |
| Bottom sheet for SVG tap reveals | Avoids tooltip clipping at edges. Always visible regardless of tap target position. Can update in-place without closing. |
| HWR-1 low-confidence popup used for model training | Turns every student interaction into a training data point. Virtuous loop: more use → better model → better recognition. |
| Send to SmartPad for mobile users | Students should always be able to solve with stylus even if they're browsing on mobile. Cross-device flow preserves the stylus-first principle. |
| Teacher annotations visible per class only | Personalises the e-library per class without breaking the shared content model. Feels like a teacher's margin notes. |
| NeuraCoins reset annually | Prevents runaway balances. Creates urgency to redeem. Academic year is the natural cycle. |

## 20. Open Questions

| Question | Impact | Recommended action |
|----------|--------|--------------------|
| Telugu TTS voice engine — which provider? Google TTS, Microsoft Azure, iSpeech, or Kaleida? | Quality of reading for Foundation/Elementary students | Evaluate 3 voices with actual Class 4–6 Telugu students. Prioritise phonetic accuracy over naturalness. |
| HWR-1 confidence threshold for showing correction popup | Too low = annoying interruptions; too high = missed training data | Start at 0.75, tune based on first 1,000 interactions |
| Problem segment quiz: drawn from Standard tier only, or mix of Foundation + Standard? | Difficulty calibration of pass gate | Recommend Standard-only for quiz (5 questions). Foundation problems remain in the segment as confidence builders. |
| Workbook tab content — is this a separate generation pass or part of the main generation pipeline? | Content volume and storage | Separate generation pass per chapter. Triggered manually by content team after Textbook is approved. |
| Notes sync — student notes are private. Does the school ever have legal right to access them? | DPDP Act 2023 compliance | Consult legal. Recommend: notes are student-only by default, with explicit consent required for any school access. |
| CSS diagram animation on tablet — auto-play or tap-to-play? | Engagement vs battery | Auto-play once, then pause. Student can replay. Consistent with desktop audit view behaviour. |
| Classwork tab — can the student print notes? | Useful for students who submit handwritten work | Print via school Wi-Fi printer (future). Export as PDF (v2). |
