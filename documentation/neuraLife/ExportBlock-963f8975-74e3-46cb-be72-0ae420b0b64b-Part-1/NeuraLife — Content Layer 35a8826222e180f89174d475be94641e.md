# NeuraLife — Content Layer

*The architecture, format, generation pipeline, age-segmentation model, and delivery system for all educational content served across the SmartPad, Teacher App, and Parent App.*

---

## 1. What the Content Layer Is

The Content Layer is not a digital library. It is the complete educational content infrastructure of NeuraLife — covering how content is sourced, generated, structured, stored, delivered, and linked to the AI intelligence layer.

Every piece of content in NeuraLife serves the closed loop:

- Student reads a concept → writes a response → GAP-1 detects an error
- Content Layer surfaces the exact visual explanation for that specific error
- Student is recommended the exact problem set that addresses that gap
- Teacher and parent see the topic the student struggled with — in plain language

Content is not passive. It is the trigger for every AI signal in the platform.

## 3. The Four Age Bands

A Class 1 student and a Class 10 student are fundamentally different cognitively, emotionally, and in motor development. The Content Layer treats them as separate product surfaces — not one product scaled up or down.

| Band | Classes | Ages | Cognitive Stage | Design Philosophy |
| --- | --- | --- | --- | --- |
| **Foundation** | 1–3 | 6–9 yrs | Concrete operational, pre-abstract | Pictures dominate. Audio everywhere. Short sessions. Play-first. |
| **Elementary** | 4–6 | 9–12 yrs | Concrete operational, pattern recognition | Illustrated content. Step-by-step logic. Rules and patterns. |
| **Middle** | 7–8 | 12–14 yrs | Early formal operational | Concept + example + practice. Abstract beginning. Social motivation. |
| **Secondary** | 9–10 | 14–16 yrs | Formal operational | Academic rigour. Exam pattern. Multi-step reasoning. Competitive context. |

## 4. Content Structure — The Five Layers

Every chapter in NeuraLife is structured in five layers. All five are generated by the Content Agent and delivered in a single `.nlc` bundle.

### Layer 1 — Concept Explanation

**The principle:** Visual first. Intuition second. Formal language third.

```
Structure per topic:
  1. Hero visual (SVG diagram or animation still)
  2. One-sentence plain-language summary
  3. Concept explanation (age-appropriate language, Flesch score calibrated to band)
  4. Key terms with definitions
  5. Textbook-level formal explanation (for exam preparation)
```

Band-specific language calibration:

| Band | Max sentence length | Vocabulary | Tone |
| --- | --- | --- | --- |
| Foundation | 8 words | Familiar, everyday words only | Story-like, warm, encouraging |
| Elementary | 12 words | Introduce new terms with simple definitions | Clear, friendly, rule-based |
| Middle | 18 words | Subject-standard terminology | Explanatory, example-led |
| Secondary | No limit | Full academic vocabulary | Precise, exam-oriented |

---

### Layer 2 — Interactive SVG Diagrams

Every textbook figure is rebuilt as an interactive SVG — not a scanned image.

**SVG generation pipeline:**

```
Content Agent (Claude) analyses the concept
  → Determines diagram type needed
  → Generates SVG code directly (clean, optimised, viewBox-based)
  → Adds interaction annotations (tap targets, hover states, reveal triggers)
  → Human auditor reviews rendered SVG in audit dashboard
  → Approved SVG committed to content library
```

**Diagram types by subject:**

| Subject | Diagram Types |
| --- | --- |
| Mathematics | Coordinate grids, geometric figures with draggable vertices, number lines, bar/pie charts |
| Physical Science | Force vector diagrams, circuit diagrams, ray optics diagrams, wave diagrams |
| Biological Science | Tap-to-reveal anatomy layers, cell structure diagrams, ecosystem maps |
| Social Studies | Zoomable geographic maps, timeline diagrams, political boundary maps |
| Chemistry | Molecular structure diagrams, periodic table excerpts, reaction equation visuals |

**SVG interaction types:**

| Interaction | Description | Implemented as |
| --- | --- | --- |
| Tap-to-reveal | Tap a structure → label + description appears | CSS class toggle |
| Layer reveal | Tap to show next layer (e.g., skin → muscle → bone) | Z-index + CSS transition |
| Hover highlight | Hover part → highlights + shows name | CSS :hover |
| Zoom region | Pinch/double-tap to zoom into diagram section | CSS transform |
| Animated state | Diagram animates between two states on tap | CSS @keyframes |

**For complex diagrams the agent cannot generate accurately** (detailed human anatomy, complex mechanical systems), the pipeline flags the topic for a human illustrator. These exceptions are tracked in the audit dashboard. Target: fewer than 10% of diagrams require human illustration.

### Layer 3 — Play-to-Learn Interactions

Concepts that must be experienced, not read. Every interaction is built as self-contained HTML/CSS/JS inside the `.nlc` bundle — no internet required.

**v1 Interaction Library (4 types — built for demo):**

**Type 1 — Tap-to-Sequence**
Student taps items in the correct order. For processes and systems.

```
Use cases: Digestive system stages, water cycle steps,
           historical event ordering, algorithm steps
Input: Touch only
GAP-1 signal: Correct sequence rate, number of attempts
```

**Type 2 — Label-the-Diagram**
Student drags labels to their correct position on a diagram.

```
Use cases: Biology anatomy, geography maps, physics apparatus,
           chemistry apparatus diagrams
Input: Touch/stylus drag
GAP-1 signal: Label accuracy, common mislabelling patterns
```

**Type 3 — Slider-Parameter**
Student moves a slider and watches a value or diagram change in real time.

```
Use cases: Ohm's Law (voltage → current), lens focal length,
           speed-distance-time, graph slope
Input: Touch drag
GAP-1 signal: Understanding of relationship direction (direct/inverse)
```

**Type 4 — Stylus Fill-the-Equation**
Student writes the missing value or step with the stylus.

```
Use cases: Mathematics working steps, chemical equation balancing,
           physics formula completion
Input: Stylus (HWR-1 processes input)
GAP-1 signal: Full error pattern classification — sign errors,
              unit errors, formula errors, calculation errors
```

**v2 Interaction Library (additions):**

| Type | Use Cases | Input |
| --- | --- | --- |
| Sketch-the-graph | Plot points, draw curves, sketch ray diagrams | Stylus — HWR-1 + GAP-1 |
| Build-the-molecule | Drag atoms to form compounds | Touch |
| Geometry-proof | Drag shapes to prove congruence/similarity | Touch + stylus |
| Circuit-builder | Connect components to build circuits | Touch |
| Simulation | Physics/chemistry virtual experiments | Touch |

**Content agent responsibility:** For each topic, the agent recommends the most appropriate interaction type from the library. Human auditor confirms or overrides. The interaction type is not pre-assigned by subject — it is assigned per concept based on what the concept requires.

### Layer 4 — Audio (Text-to-Speech)

Audio read-aloud is available for every grade. Primary focus is Foundation and Elementary bands.

**Implementation:**

```
Technology: Android Text-to-Speech (on-device, offline)
Languages: Telugu + English
Voices: Google TTS voice pack (Telugu female voice, English neutral)
Trigger: Speaker icon on every text block, every problem statement,
         every feedback message
```

**Per-band audio behaviour:**

| Band | Audio behaviour |
| --- | --- |
| Foundation (1–3) | Auto-plays concept summary on topic open. Every problem read aloud automatically. Alphabet and word pronunciation mode with phonetic breakdown. |
| Elementary (4–6) | Audio available on tap. Key terms read with pronunciation on first encounter. |
| Middle (7–8) | Audio on tap only. Problem statements read aloud on request. |
| Secondary (9–10) | Audio on tap. Useful for Telugu medium exam term pronunciation. |

**Foundation — Alphabet and Pronunciation Mode:**

For Classes 1–2, a dedicated pronunciation mode exists:

- Student writes a letter or word with stylus
- HWR-1 recognises it
- TTS pronounces the word clearly
- Phonetic breakdown displayed: "CAT → /k/ /æ/ /t/"
- Student can tap each phoneme to hear it in isolation

This is the foundational literacy feature — reading and writing taught together on one device.

### Layer 5 — Problem Sets

Three difficulty tiers per topic. Every problem is tagged to exact GAP-1 error patterns.

**Problem structure:**

```json
{
  "problem_id": "PS-MATH-10-QE-F-001",
  "subject": "MATHEMATICS",
  "class_year": 10,
  "topic": "QUADRATIC_EQUATIONS",
  "difficulty": "FOUNDATION",
  "medium": "ENGLISH",
  "input_type": "STYLUS",
  "problem_text": "Solve for x: x² - 5x + 6 = 0",
  "problem_audio": true,
  "expected_method": "FACTORING",
  "target_error_patterns": [
    "SIGN_ERROR_FACTORING",
    "INCORRECT_FACTOR_PAIRS",
    "MISSING_BOTH_SOLUTIONS"
  ],
  "solution_steps": [
    "Factor the equation: (x - 2)(x - 3) = 0",
    "Apply zero product property",
    "x = 2 or x = 3"
  ],
  "hint_sequence": [
    "What two numbers multiply to give +6 and add to give -5?",
    "Try -2 and -3. Check: (-2) × (-3) = 6, (-2) + (-3) = -5",
    "Write it as (x - 2)(x - 3) = 0 and solve each bracket"
  ],
  "remedial_content_ref": "MATH-10-CH3-TOPIC2"
}
```

**Difficulty tiers:**

| Tier | Description | Purpose |
| --- | --- | --- |
| **Foundation** | Single-step. Builds confidence. Direct application of one rule. | Entry point — everyone can do this |
| **Standard** | Textbook-style. Multi-step. Exam pattern. | Core learning + exam preparation |
| **Advanced** | Application in new context. Multi-concept. Requires synthesis. | Stretch — for MASTERED students |

**Hint system:**
Three hints available per problem, revealed on student request. Each hint is one step closer to the answer — not the full solution. Third hint reveals the method without the final answer. Hint usage is tracked by GAP-1 as `hint_dependency_rate`.

**Cross-grade prerequisite linking:**
Every topic in the problem set is tagged with prerequisite topics from earlier classes. When GAP-1 detects a gap, the system surfaces prerequisite content automatically:

```
Student error on: Quadratic Equations (Class 10, MATH)
GAP-1 detects: FACTORING_FAILURE
Prerequisite check: LINEAR_EQUATIONS (Class 8), FACTORS_AND_MULTIPLES (Class 6)
System action: Surfaces Class 8 Chapter 2 → "Let's revisit this first"
```

## 6. Content Navigation

```
Grade Selection
  → Current enrolled grade opens by default
  → Student can switch to any grade (other grades = on-demand chapter download)

Subject Selection
  → Subjects for selected grade and board

Chapter Selection
  → All chapters listed with completion status
  → Completion: % of topics attempted + mastery indicator
  → Recommended chapters highlighted (from Recommendation Engine)

Topic / Segment / Exercise Selection
  → All topics in the chapter listed
  → Each topic shows: concept, diagram, interaction, problems
  → Topics can be navigated independently — not forced linear
  → "Start from beginning" option for new chapters
```

**Progress indicators per chapter:**

| Indicator | Meaning |
| --- | --- |
| 🔵 Blue ring | Started, in progress |
| ✅ Green fill | Completed (all topics attempted) |
| ⚠️ Amber dot | Flagged — GAP-1 detected persistent gap |
| ⭐ Star | Mastered — calibrated score above threshold |
| 🔒 Lock (Foundation only) | Complete previous topic to unlock |

Foundation band (Classes 1–3) uses sequential locking — students must complete topics in order to build foundational skills. All other bands: fully open navigation.

## 8. The Content Agent Pipeline

### Architecture

```
INPUT: Textbook chapter — PDF, image scan, or text extract
         ↓
┌─────────────────────────────────────────────────────┐
│              Stage 1 — Extraction Agent             │
│  Model: Claude (claude-sonnet-4-20250514)            │
│  Task: Parse chapter → structured JSON chapter map  │
│  Output:                                            │
│    { title, topics: [{ name, subtopics, formulas,  │
│      key_terms, examples, textbook_page_refs }] }   │
└──────────────────────┬──────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────┐
│           Stage 2 — Content Generation Agent        │
│  Model: Claude (claude-sonnet-4-20250514)            │
│  Prompt includes: band, board, medium, subject      │
│  For each topic generates:                          │
│    • Concept explanation (band-calibrated)          │
│    • One-sentence summary                           │
│    • SVG diagram code (clean, viewBox-based)        │
│    • Animation storyboard (CSS @keyframes spec)     │
│    • Play-to-learn interaction recommendation       │
│    • 10 problems × 3 difficulty tiers               │
│    • Error pattern tags per problem                 │
│    • Hint sequences (3 hints per problem)           │
│    • Prerequisite topic tags                        │
│    • YouTube search query                           │
│    • Audio TTS text (plain, no markdown)            │
└──────────────────────┬──────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────┐
│              Stage 3 — Human Audit Dashboard        │
│  Interface: Web-based internal tool                 │
│  Auditor types:                                     │
│    • Subject expert (accuracy review)               │
│    • Telugu language expert (medium accuracy)       │
│    • Teacher reviewer (age-appropriateness)         │
│  Per topic, auditor can:                            │
│    • Approve as-is                                  │
│    • Edit text inline                               │
│    • Reject SVG → flag for human illustrator        │
│    • Override interaction type                      │
│    • Approve/replace YouTube link                   │
│    • Reject and regenerate (with feedback note)     │
│  Nothing publishes without audit approval           │
└──────────────────────┬──────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────┐
│              Stage 4 — Rendering Pipeline           │
│  SVG code → validated, optimised SVG files          │
│  Animation spec → CSS @keyframes files              │
│  Interactions → HTML/CSS/JS bundles                 │
│  Problems → validated JSON problem sets             │
│  Audio text → TTS-ready plain text strings          │
│  All assets → packaged into .nlc bundle             │
└──────────────────────┬──────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────┐
│              Stage 5 — Device QA                    │
│  .nlc bundle loaded on test SmartPad                │
│  Checklist:                                         │
│    □ Renders correctly on E-Ink (v2) and LCD (v1)  │
│    □ All interactions function offline              │
│    □ Stylus input processed by HWR-1               │
│    □ Audio TTS plays correctly in Telugu + English  │
│    □ Bundle size within storage budget              │
│  Sign-off → Published to content_library table      │
└─────────────────────────────────────────────────────┘
```

## 10. Content Delivery

### SmartPad — On-Device Rendering

Content renders in a sandboxed **WebView** (v1, stock Android tablet) or **native Kotlin renderer** (v2, NeuraOS E-Ink).

The SmartPad app extracts the `.nlc` zip to a local directory and serves content from `file://` URI — no network calls during content display. All SVGs, CSS, JS run locally.

```
Student opens Chapter 3:
  → App checks cached_content table: MATH-10-CH3 present?
    → YES: Extract .nlc → render topic list → student navigates
    → NO (different grade, on-demand):
        → Show download prompt (size + estimated time)
        → Student confirms → Download from Supabase Storage
        → Verify checksum → Extract → Open
```

### Renderer responsibilities:

```kotlin
// SmartPad ContentRenderer (Kotlin)

fun renderTopic(topicDir: File, band: Ageband) {

  // 4. Load interaction/index.html → render in sandboxed WebView
  // 5. Inject HWR-1 stylus listener for STYLUS_INPUT type interactions
  // 6. Load problems.json → render problem UI
  // 7. Register GAP-1 listener on problem submission
  // 8. Enable TTS button with audio_text from content.json
}
```

## 11. Content–AI Integration

This is the feature that separates the Content Layer from any other EdTech platform.

### Problem → GAP-1 → Content Feedback Loop

```
Student writes answer with stylus
  → HWR-1 recognises text
  → GAP-1 compares to expected method and solution
  → Error pattern classified: SIGN_ERROR_FACTORING
  → Content Layer looks up: which topic addresses SIGN_ERROR_FACTORING?
  → Surfaces: specific section of "Moving Terms" animation from Chapter 2
  → "Let's look at this again:" → shows the relevant SVG + animation
  → Gives 3 new problems targeting SIGN_ERROR_FACTORING specifically
```

### Recommendation → Content Delivery Loop

```
ML Pipeline 3 (Recommendation Engine) computes:
  → Student NID-2025-AP-084291 has persistent gap: FACTORS_AND_MULTIPLES (Class 6)
  → content_recommendations record created
  → On next sync: recommendation pushed to SmartPad
  → SmartPad downloads: MATH-6-CH4.nlc (on-demand, background)
  → Student sees: "Recommended for you" card on home screen
  → Student opens it → studies → GAP-1 signals improvement
  → Next calibration: prerequisite gap resolved
```

### Teacher Push → Content Delivery

```
Teacher identifies class-wide gap in Physical Science
  → POST /api/v1/teacher/push-resource
      { class: "10-A", content_ids: ["PHYS-10-CH2"], problem_set_ids: [...] }
  → API creates pending_recommendations for all students in 10-A
  → On next sync: each SmartPad downloads the pushed content
  → Content appears in student's "From Your Teacher" section
  → Completion tracked per student → visible in Teacher App
```

## 12. Foundation Band — Special Design Rules

Classes 1–3 require a fundamentally different design philosophy.

### Alphabet and Writing Mode

Available in Telugu and English for Classes 1–2:

```
Student mode: ALPHABET_PRACTICE
  → Letter shown large on screen (with stroke order guide)
  → Student traces with stylus (overlaid on guide)
  → HWR-1-F evaluates: stroke order, formation quality, proportions
  → Audio TTS: pronounces letter + example word
  → Feedback: animated star + "Well done!" (not score percentage)
  → Next letter unlocks on 3 correct traces
  → Parent App: shows which letters mastered this week
```

### Foundation UI Rules

| Element | Rule |
| --- | --- |
| Font size | Minimum 18px body, 24px headings |
| Colours | Bright primary palette. High contrast. Each subject has a colour. |
| Navigation depth | Maximum 3 taps from home to any content |
| Icons | Illustrated characters (animal mascots per subject), not abstract icons |
| Progress | Stars and stickers, not percentages |
| Error feedback | "Oops! Let's try again! 🌟" — never show red X or failure language |
| Session auto-pause | 20 minutes of active use → "Time for a break!" screen |
| Topic locking | Sequential — must complete each topic to unlock the next |
| Audio | Auto-plays on topic open. Every problem read aloud automatically. |

## 15. Confirmed Decisions

| Decision | Rationale |
| --- | --- |
| Claude-powered Content Agent with human audit gate | Scale requires AI generation. Quality requires human review. Neither alone is sufficient. |
| SVG generated as code by Claude, not from design tools | Automated, scalable, version-controlled. No design tool dependency. Complex exceptions flagged for human illustrator. |
| .nlc bundle format (ZIP-based) | Self-contained, offline-capable, checksummed, version-aware. Existing tools handle ZIP on all platforms. |
| WebView renderer for v1, native Kotlin for v2 (E-Ink) | WebView is fast to build. Native renderer needed for E-Ink partial refresh performance. |
| Four age bands with fully separate design rules | A Class 1 and Class 10 student are different products, not different sizes of the same product. |
| Dual input: stylus for writing, touch for interaction | Stylus is non-negotiable for writing analysis. Touch is better for play-to-learn. Both coexist. |
| HWR-1-F separate model for Foundation band | Class 1–3 strokes are fundamentally different from developed handwriting. One model cannot serve both. |
| Audio TTS on every grade, auto-play on Foundation | Pronunciation and phonetics are foundational literacy features. Available everywhere. |
| Sequential topic locking for Foundation only | Foundational skills must be built in order. All other bands: open navigation. |
| Demo content: Classes 9–10, SCERT AP, 4 subjects | Sufficient to demonstrate the complete content experience. Expand within 6 months. |
| YouTube: curated per topic, not auto-embedded | AI-suggested query + human curator selects the best video. Quality gate before any video reaches students. |
